# 常态化部署规范（Mac Studio 专用）

> 来源项目：SOTAgent
> 总结日期：2026-04-13
> 标签：部署, launchd, Mac Studio, 后台服务, 端口管理
> 适用场景：需要在 Mac Studio 上配置后台常驻服务时

本规范适用于 **Mac Studio** 设备上需要后台常驻的应用。
**MacBook 不适用** — MacBook 上的项目按需手动启动，不配置 launchd 自启动。

---

## 1. 适用条件

满足以下任一条件的应用应执行常态化部署：
- 提供 HTTP/WebSocket 服务（API 端口对外暴露）
- 需要定时执行后台任务（巡检、同步、监控）
- 是其他应用的依赖（如 PolarPrivate 是 PolarClaw 的凭证来源）
- 用户明确要求"后台运行"

---

## 2. 部署四件套

每个常态化部署的应用必须完成以下四步：

### 2a. launchd 自启动

创建 `~/Library/LaunchAgents/com.<project>.plist`：

```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "...">
<plist version="1.0">
<dict>
    <key>Label</key>
    <string>com.<project>.<service></string>
    <key>ProgramArguments</key>
    <array>
        <string>/bin/bash</string>
        <string>-c</string>
        <string>cd <project-dir> &amp;&amp; <project-dir>/scripts/run-with-update.sh</string>
    </array>
    <key>WorkingDirectory</key>
    <string><project-dir></string>
    <key>RunAtLoad</key>
    <true/>
    <key>KeepAlive</key>
    <true/>
    <key>StandardOutPath</key>
    <string>~/Library/Logs/<project>.log</string>
    <key>StandardErrorPath</key>
    <string>~/Library/Logs/<project>.log</string>
    <key>EnvironmentVariables</key>
    <dict>
        <key>PATH</key>
        <!-- nvm Node 路径放最前面，避免 Homebrew Node 版本不匹配 native 模块 -->
        <string>~/.nvm/versions/node/v20.20.2/bin:/opt/homebrew/bin:/usr/local/bin:/usr/bin:/bin</string>
        <key>HOME</key>
        <string>~</string>
    </dict>
    <key>ThrottleInterval</key>
    <integer>30</integer>
</dict>
</plist>
```

### 2b. SOTAgent 端口备案

所有 HTTP/WebSocket 端口必须在 SOTAgent 的 `port_registry` 表中注册：

```bash
RESOURCES_DB="$HOME/Polarisor/SOTAgent/data/resources.sqlite"
sqlite3 "$RESOURCES_DB" "
INSERT INTO port_registry (port, service_name, project, device_id, status, allocated_at, last_verified)
VALUES (<PORT>, '<service-name>', '<project>', 'mac-studio', 'active', datetime('now'), datetime('now'))
ON CONFLICT(port) DO UPDATE SET service_name='<service-name>', project='<project>', status='active', last_verified=datetime('now');
"
```

### 2c. 版本监控与自动更新

每个项目的 `scripts/run-with-update.sh` 负责：
1. 设置正确的 Node 版本（通过 `.nvmrc` 或默认 v20）
2. `git fetch origin main` — 检查远程是否有更新
3. 如果 local HEAD != origin/main 且无本地改动 → `git pull` + `npm install`
4. `exec node start.mjs`（或其他启动命令）

launchd `KeepAlive=true` + `ThrottleInterval=30` 确保：
- 进程崩溃后 30 秒内自动重启
- 重启时自动执行版本检查
- 相当于"每次启动时自动部署最新版本"

**注意**：如果有本地未提交的改动，跳过自动 pull，避免覆盖开发中的代码。

### 2d. 日志

统一使用 `~/Library/Logs/<project>.log`。

查看日志：`tail -f ~/Library/Logs/<project>.log`

## 4. 常用运维命令

```bash
# 查看所有 launchd 服务
launchctl list | grep -E 'sotagent|privportal|polarclaw'

# 重启某个服务
launchctl unload ~/Library/LaunchAgents/com.polarclaw.agent.plist
launchctl load ~/Library/LaunchAgents/com.polarclaw.agent.plist

# 查看所有 SOTAgent 注册端口
sqlite3 ~/Polarisor/SOTAgent/data/resources.sqlite \
  "SELECT port, service_name, project FROM port_registry WHERE status='active' ORDER BY port"

# 查看所有日志
ls ~/Library/Logs/*.log

# 强制更新并重启某个服务
cd ~/Polarisor/<project> && git pull && npm install
launchctl unload ~/Library/LaunchAgents/com.<project>.plist
launchctl load ~/Library/LaunchAgents/com.<project>.plist
```

## 5. Node 版本陷阱

launchd 环境不会加载 `~/.bashrc` / `~/.zshrc`，因此 nvm 不可用。
必须在 plist 的 `PATH` 中显式指定 nvm Node 路径。

如果 native 模块（如 better-sqlite3）报 `NODE_MODULE_VERSION` 错误：
1. 确认 plist PATH 中的 Node 版本与 `npm rebuild` 时使用的版本一致
2. `npm rebuild better-sqlite3`（在正确的 Node 版本下）
3. 重启 launchd 服务

## 6. 设备约定

| 设备 | 常态化部署 | 说明 |
|------|-----------|------|
| **Mac Studio** | ✅ 必须 | 所有常驻服务、后台任务在此设备运行 |
| **MacBook** | ❌ 不配置 | 移动设备，按需手动启动，不配置 launchd 自启动 |

此约定写入各 Agent 的上下文中：
- 如果 `hostname` 包含 "Studio" 或设备ID 为 "mac-studio" → 执行常态化部署
- 如果 `hostname` 包含 "MacBook" → 不配置 launchd
